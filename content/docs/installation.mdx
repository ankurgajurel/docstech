# Installation

## Prerequisites

- **Node.js 18.17+** — LTS recommended. Verified on 20.x and 22.x.
- **Bun 1.0+** — Used as the default package manager.
- **Git 2.30+** — Required for cloning the repository.

## Clone the Repository

```bash
git clone https://github.com/ankurgajurel/docstech.git
cd docstech
```

## Install Dependencies

```bash
bun install
```

## Start the Development Server

```bash
bun dev
```

Your docs site is now running at `http://localhost:3000` with hot module replacement. Content changes are reflected instantly without a full rebuild.

```
docstech/

```

## Writing Content

All your documentation lives in `content/docs/`. Create or edit `.mdx` files there. Each file needs frontmatter with a `title` and `description`:

```mdx
---
title: Webhooks
description: Configure event-driven notifications for deployment and content lifecycle events.
---

# Webhooks

Subscribe to real-time events from the DocsTech platform.
Webhooks are delivered as POST requests with a signed payload.
```

That's it — the build pipeline picks up new files, adds them to the navigation tree, and renders them as fully styled pages.

### URL Routing

File paths inside `content/docs/` map directly to URL routes:

- `content/docs/introduction.mdx` → `/docs/introduction`
- `content/docs/api/rest.mdx` → `/docs/api/rest`
- `content/docs/guides/webhooks.mdx` → `/docs/guides/webhooks`

### Sidebar Navigation

To add pages to the sidebar, edit the navigation config at `components/sidebar/navigation.ts`. Group pages by section and assign icons.

## Production Build

```bash
bun run build
```

The output is a static export ready for deployment to any CDN, edge network, or container runtime.
